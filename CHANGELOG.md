# Changelog

## 0.3.31

- `createInMemoryAdapter()` — a complete in-memory `StorageAdapter` shipped with the package; the quick start is now fully self-contained.
- Docs: correct package name in every code sample, install instructions, stated reading order, fixed documentation links, provenance note.
- Exports map: `types` condition ordered first; `./package.json` exposed.
- Foundation world types genericized (`NutritionWorld`, `FinanceWorld`, `InboxWorld`, `FitnessWorld`).

## 0.3.30 — initial public release

Extracted from a private foundation that runs five production persona apps. Public history starts here; earlier versions (0.1.x–0.3.29) lived in the private lab. Highlights of the foundation as published: scenario-first prompt assembly, CRUD-native entities and memory with an LLM review lifecycle (fact-fidelity guarded), built-in EQ primitives, turn traces with request fingerprints, offline prompt-surface tests and live persona evals, and prompt audits that detect over-prescription.
